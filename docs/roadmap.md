# Roadmap

## P0：現況盤點與文件化

整理前台、後台、使用對象、系統邊界、production 保護規則與初始 GitHub 紀錄。

## P1：建立 Docker skeleton

建立 `Dockerfile`、`docker-compose.yml`、`.env.example`、web skeleton、預留 db 與部署規劃。

## P2：建立本機開發環境

決定技術棧、安裝開發工具、建立 lint/test/build 流程與本機資料庫初始化方式。

## P3：資料表設計

設計會友、同工、通訊錄、課程活動、報名、繳費、上課狀況、報到與打卡紀錄資料表。

## P4：API 設計

定義前台報到、QR Code、手動搜尋、同工打卡、後台查詢、管理與匯出 API。

## P5：前台重建

重建聚會類型選擇、課程聚會選擇、牧區小組選擇、所屬單位選擇、QR Code 掃描與手動搜尋報到。

## P6：後台重建

重建打卡紀錄查詢、課程活動管理、員工名單管理、通訊錄管理、報名清單、繳費狀態與上課狀況。

## P7：匯出 Excel

建立可追蹤、可權限控管的 Excel 匯出流程，並確認欄位、格式與篩選條件。

## P8：權限與登入安全

設計管理者登入、角色權限、同工打卡安全碼、安全記錄與匯出限制。

## P9：測試與備份

建立單元測試、整合測試、Docker 啟動測試、資料庫備份與還原演練。

## P10：正式部署規劃

規劃正式部署環境、reverse proxy、SSL、volume、備份、監控、rollback 與切換步驟。未有明確指令前不得部署到 `.240`。
